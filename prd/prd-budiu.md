# 布丢 — 智能宠物项圈 App PRD

## 产品概述

### 产品愿景
布丢是一款面向中国城市养宠人群的智能宠物项圈配套 App，通过"硬件数据 + AI"为宠物主人提供安全守护、社交互动和健康管理三位一体的服务。

### 一句话定位
**让上班族安心养宠的智能项圈平台。**

### 产品路线
- **Phase B（MVP）**: 安全守护 — 利用现有硬件，解决"上班时宠物无人看管"的核心痛点
- **Phase C**: 数据驱动社交 — 基于项圈数据打造差异化宠物社区
- **Phase A**: AI 健康管家 — 自建 AI 模型，提供运动分析和健康预测

### 现有硬件能力
布丢项圈已具备：GPS 实时定位、电子围栏、声光功能（声波召回 + LED 指示灯）、加速度传感器（运动数据）、蓝牙连接。

## 目标用户

### 核心用户画像

**主要人群：城市独居养狗青年**
- 年龄 25-35 岁，90后/00后（占养宠人群 70%）
- 女性为主（63%），一线/新一线城市
- 独居或与伴侣同住，工作时间长（朝九晚六以上）
- 月均宠物花费 ¥500-2000，愿意为宠物安全付费
- 活跃于小红书/抖音宠物内容

**核心痛点（按严重程度排序）：**
1. "上班或出差时无人照顾宠物"（38.4% 宠物主人第一痛点）
2. 遛狗时担心宠物跑丢（尤其是新手养宠者）
3. 不了解宠物日常活动状态（"它今天过得怎么样？"）
4. 找不到附近的遛狗伙伴

### 次要人群
- 家有老人帮忙遛狗的上班族（想远程监控遛狗过程）
- 刚养宠物的新手铲屎官（焦虑型，对安全需求更强）

## Phase B：安全守护（MVP 详细规格）

### 目标
让用户上班时随时掌握宠物位置和状态，在异常情况下即时响应。

### 核心指标
| 指标 | 目标值 | 说明 |
|------|--------|------|
| 项圈激活率 | >80% | 购买项圈后完成 App 绑定的比例 |
| DAU/MAU | >30% | 日活跃用户/月活跃用户 |
| 围栏告警准确率 | >95% | 触发告警且确实越界的比例 |
| 告警推送延迟 | <30秒 | 从触发到用户收到通知 |
| 次月留存率 | >50% | 首月激活用户第二月仍活跃 |

### 功能清单

#### P0 — 必须上线

**F1: 宠物档案**
- 创建宠物档案：昵称、头像、品种、生日、体重、性别、绝育状态
- 支持多宠物管理（一个账号绑定多只宠物/多个项圈）
- 疫苗记录（手动录入：疫苗名称、接种日期、下次接种提醒）

**F2: 项圈绑定与设备管理**
- 蓝牙扫描 → 项圈配对 → WiFi/4G 联网配置
- 项圈电量实时显示 + 低电量推送提醒（<20% 和 <10% 两档）
- 固件 OTA 升级
- 项圈状态：在线/离线/充电中

**F3: GPS 实时定位**
- 地图上显示宠物实时位置（接入高德地图 SDK）
- 支持切换地图模式（标准/卫星）
- 位置刷新频率：
  - 正常模式：每 5 分钟上报一次
  - 追踪模式（手动开启或围栏触发）：每 10 秒上报一次
- 定位精度目标：室外 <10m，市区 <30m

**F4: 电子围栏**
- 在地图上画圆形围栏（设置中心点 + 半径，最小 50m，最大 5km）
- 支持最多 5 个围栏同时生效
- 宠物离开围栏 → 即时推送告警（App 推送 + 短信备选）
- 宠物返回围栏 → 推送"已安全返回"
- 围栏支持启用/禁用/按时段生效（如工作日 9:00-18:00）

**F5: 遛狗轨迹记录**
- 手动开始/结束遛狗记录
- 记录：轨迹路线、距离、时长、时间戳
- 遛狗结束后展示轨迹概览卡片（地图 + 数据摘要）
- 历史遛狗记录列表（按日期查看）

**F6: 声光远程控制**
- App 一键触发项圈声音（用于召回）
- App 一键触发项圈 LED 灯（夜间可视）
- 声音可选模式：短促蜂鸣 / 连续鸣响

**F7: 活动异常告警**
- 长时间无运动告警：项圈加速度传感器连续 N 小时无显著运动 → 推送提醒（N 可配置，默认 4 小时，8:00-22:00 生效）
- 告警推送支持：App 推送 + 微信服务号消息

**F8: 通知中心**
- 统一管理所有告警和系统消息
- 告警分类：围栏告警、活动异常、设备告警（低电量/离线）
- 已读/未读状态
- 告警历史记录

#### P1 — 优先上线

**F9: 家庭成员共享**
- 邀请家人（通过微信分享邀请链接）共同管理同一只宠物
- 所有成员可查看位置、轨迹、收到告警
- 主账号可管理成员权限

**F10: 活动数据概览**
- 每日活动量统计（基于加速度传感器：活跃时间 / 休息时间）
- 按天/周的活动趋势图表
- 活动数据与遛狗记录关联

**F11: 微信生态接入**
- 微信小程序：轻量版（查看位置、收告警、声光控制）
- 微信服务号：告警消息推送通道
- 微信登录

#### P2 — 后续迭代

**F12: 宠物寻回辅助**
- 项圈离线后，记录最后已知位置
- 进入"寻回模式"：高频声光信号 + 显示最后位置 + 附近蓝牙探测（需靠近）
- 生成走失信息卡片（含照片、品种、最后位置），可一键分享到微信/朋友圈

**F13: 智能围栏建议**
- 基于遛狗轨迹数据，AI 自动建议常去区域的围栏设置
- 技术方案：统计学聚类（DBSCAN），无需大模型

**F14: 遛狗定时提醒**
- 基于用户设置或历史遛狗规律，提醒"该遛狗了"
- 支持按星期自定义提醒时间

### 非功能需求

**性能**
- App 冷启动 < 3 秒
- 地图加载 < 2 秒
- 告警推送端到端延迟 < 30 秒

**兼容性**
- iOS 14+ / Android 10+
- 适配主流机型（iPhone 12 及以上、主流 Android 品牌近 3 年旗舰和中端机型）

**安全与隐私**
- 位置数据加密传输和存储
- 用户数据符合《个人信息保护法》要求
- 位置数据默认仅对宠物主人和共享家庭成员可见

**国际化**
- 本阶段仅支持简体中文
- 架构预留多语言支持能力

## Phase C：数据驱动社交（概要）

### 目标
在 B 阶段积累的用户和项圈数据基础上，构建以硬件数据为差异化壁垒的宠物社区，提升用户粘性并开辟变现渠道。

### 前置条件
- Phase B 上线运营，项圈激活量达到一定规模（建议 >5000 台）
- 用户遛狗轨迹和活动数据积累 >3 个月

### 核心功能（概要）

**AI 宠物日记**
- 基于项圈每日运动/睡眠/位置数据，AI 自动生成拟人化日记
- 示例："今天小布跑了2.8公里，在朝阳公园遇到了3个伙伴，下午睡了个长觉，活力值85分！"
- 技术方案：通义千问/DeepSeek API + prompt 工程
- 用户可编辑、配图后分享到社区或微信

**遛狗热力图 + 附近宠友**
- 基于匿名化的 GPS 轨迹数据，生成本地遛狗热力图（"你附近最受欢迎的遛狗路线"）
- 匹配遛狗时间和路线相近的宠物主人，推荐"附近宠友"
- 支持发起遛狗邀约

**运动排行榜 / 挑战赛**
- 基于项圈活动数据的"本周遛狗排行"、"减肥挑战"等游戏化功能
- 区域排行榜（基于城市/社区）

**宠物档案页**
- 数据驱动的宠物"主页"：活动统计 + 遛狗地图 + 成长记录
- 不是照片墙，而是有数据故事的宠物简历

### 关键原则
- 社交必须依托项圈数据差异化，不做"又一个宠物 Instagram"
- 社区是变现手段（电商导购佣金 + 品牌合作），不是目的

### 商业模式
- 电商导购佣金（宠物食品/用品 CPS）
- 品牌合作（宠物品牌社区营销）
- 增值功能订阅（高级社交功能）

## Phase A：AI 健康管家（概要）

### 目标
基于 B/C 阶段积累的大量项圈数据，自建 AI 模型为用户提供宠物运动分析和健康趋势预测，实现订阅制变现。

### 前置条件
- 项圈数据积累量达到可训练规模（建议 >10万条日活动记录）
- AI 工程师团队到位，完成基座模型选型和微调实验

### 核心功能（概要）

**运动模式 AI 分析**
- 基于项圈加速度 + GPS 数据，识别运动类型（散步/奔跑/休息/玩耍）
- 对比品种基准数据，判断活动量是否正常
- 技术方案：自有数据 + 开源基座模型(Qwen/DeepSeek) + LoRA 微调

**健康趋势预测**
- 基于长期活动/睡眠数据的趋势分析，识别异常模式
- 示例："小布最近一周活动量下降 30%，睡眠时间增加，建议关注"
- 重要边界：提供"健康趋势观察"，不做"医疗诊断"

**LLM 宠物顾问**
- 基于 RAG（宠物知识库 + 公开兽医文献）的问答助手
- 用户可拍照/文字咨询宠物行为和养护问题
- 技术方案：开源 LLM + RAG，不接入宠智灵（数据主权和竞争风险考量）

**宠物健康日报**
- 整合运动分析 + 睡眠数据 + 活动异常 → 每日健康评分和报告
- 周报/月报趋势对比

### AI 技术策略
```
自建为主，各阶段渐进：
B 阶段 → 规则引擎 + 简单统计异常检测（无需大模型）
C 阶段 → 通义千问/DeepSeek API + prompt 工程（日记生成/推荐）
A 阶段 → 开源基座模型 + LoRA 微调（自有项圈数据）+ RAG（宠物知识库）
```

### 商业模式
- App 订阅：¥15-30/月（健康报告 + AI 顾问）
- 年订阅折扣：¥128-268/年

## 技术架构概要

### 整体架构

```
┌─────────────┐     ┌──────────────┐     ┌──────────────────┐
│  布丢项圈    │────▶│  IoT 接入层   │────▶│   后端服务        │
│ (GPS/BLE/4G)│     │ (MQTT Broker) │     │  (业务逻辑/API)   │
└─────────────┘     └──────────────┘     └────────┬─────────┘
                                                   │
                    ┌──────────────┐     ┌────────▼─────────┐
                    │  布丢 App     │◀───│   API Gateway     │
                    │ (iOS/Android) │     └──────────────────┘
                    └──────┬───────┘               │
                           │              ┌────────▼─────────┐
                    ┌──────▼───────┐     │   数据存储层       │
                    │  微信小程序   │     │ (时序DB/关系DB/OSS)│
                    └──────────────┘     └──────────────────┘
```

### 关键技术选型（待技术架构阶段确认）

| 层级 | 建议方向 | 说明 |
|------|---------|------|
| App 端 | Flutter 或 React Native | 双端同时开发，减少人力 |
| 后端 | Go / Java | 高并发 IoT 数据接入 |
| IoT 接入 | MQTT + EMQX | 项圈数据上报标准协议 |
| 时序数据库 | TDengine / InfluxDB | 存储 GPS 轨迹和传感器数据 |
| 地图 | 高德地图 SDK | 中国市场合规，定位精度好 |
| 推送 | 个推 / 极光 + 微信模板消息 | 多通道告警推送 |
| 对象存储 | 阿里云 OSS | 用户头像、宠物照片等 |

### 数据架构核心原则
- **位置数据本地优先**：位置数据存储在国内服务器，符合数据合规要求
- **时序数据分层**：热数据（7天）→ 温数据（90天）→ 冷数据（归档）
- **为 AI 预留**：从 B 阶段开始，原始传感器数据完整保存，为 C/A 阶段的 AI 训练积累数据

## 风险与应对

| 风险 | 严重程度 | 应对策略 |
|------|---------|---------|
| 项圈续航不足导致用户流失 | 高 | 正常模式低功耗设计（5分钟上报），追踪模式限时自动关闭；App 明确提示电量 |
| 告警不准确（误报/漏报） | 高 | 围栏设置最小 50m 避免 GPS 漂移误触；告警前连续 2 次确认位置；提供灵敏度设置 |
| GPS 室内/地下定位失效 | 中 | 结合 WiFi/基站辅助定位；明确告知用户室内定位限制 |
| 小红书/抖音宠物内容挤压社区（C 阶段） | 中 | 社交功能绑定项圈数据差异化（AI 日记、遛狗匹配），不做通用内容社区 |
| 健康建议引发法律风险（A 阶段） | 中 | 只做"健康趋势观察"不做"医疗诊断"，界面明确免责声明 |
| 宠智灵推出 C 端竞品 | 低 | AI 自建路线保证数据主权和差异化；宠智灵强项在 B2B，C 端执行力待验证 |
